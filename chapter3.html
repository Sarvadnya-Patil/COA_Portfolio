1. Table of Operations (ALU Function Mappings):

The ALU is designed as an 8-bit execution unit. A 4-bit Select/Opcode bus chooses the active operation as detailed below:

Opcode Mnemonic Operation Description / Mathematical Definition
0x00 LOAD A $Y = A$ Passes input $A$ directly to output $Y$. Registers state change flags (Z, S).
0x01 STORE A $Y = A$ Passes operand $A$ to output bus to be buffered by primary memory data lines.
0x02 ADD $Y = A + B$ Computes 8-bit binary addition. Updates Zero (Z), Carry (C), Sign (S), and Overflow (OV) status flags.
0x03 SUB $Y = A - B$ Computes 8-bit binary subtraction. Updates flags. Carry (C) acts as borrow-out.
0x04 MUL $Y = (A \times B) \pmod{256}$ Performs unsigned multiplication. Truncates result to the lower 8 bits. Sets Z and S flags.
0x05 DIV $Y = \lfloor A / B \rfloor$ Performs unsigned division. If $B = 0$, registers division-by-zero error, and output remains $0$ with overflow active.
0x06 AND $Y = A \text{ AND } B$ Computes bitwise logical AND operation ($Y_i = A_i \cdot B_i$).
0x07 OR $Y = A \text{ OR } B$ Computes bitwise logical OR operation ($Y_i = A_i + B_i$).
0x08 XOR $Y = A \text{ XOR } B$ Computes bitwise logical XOR operation ($Y_i = A_i \oplus B_i$).
0x09 2's COMP $Y = \overline{A} + 1$ Calculates the 2's complement (negation) of input $A$ by inverting all bits and adding $1$.
0x0A COMPARE $A - B$ (flags only) Compares $A$ and $B$. Status flag bits are set ($Z=1$ if $A=B$, $S=1$ if $A < B$, etc.). Results bus remains $0$.
0x0B SHL, #n $Y = A \ll n$ Logical Shift Left: shifts operand $A$ to the left by $n$ bits. LSB positions are filled with zeros.
0x0C SHR, #n $Y = A \gg n$ Logical Shift Right: shifts operand $A$ to the right by $n$ bits. MSB positions are filled with zeros.
0x0D ASR, #n $Y = A \text{ ASR } n$ Arithmetic Shift Right: shifts operand $A$ to the right by $n$ bits, replicating the sign bit ($A_7$) in all shifted MSB spots.

2. Summary of ALU Flag Operations:

  • Zero Flag (Z): Evaluated using a NOR gate combining all output bits ($Z = \overline{Y_7 + Y_6 + \dots + Y_0}$). Evaluates to $1$ if the result is exactly $0$.
  • Carry Flag (C): Set when an addition generates a carry out of the MSB ($C_8$), or when a subtraction requires a borrow out of the MSB. For shifts, it caches the last shifted-out bit.
  • Sign Flag (S): Set equal to the MSB of the output result ($S = Y_7$), representing negative results in 2's complement notation.
  • Overflow Flag (OV): Indicates signed arithmetic overflow. For addition, it is set if adding two positive numbers yields a negative, or two negative numbers yields a positive: $OV = A_7 \cdot B_7 \cdot \overline{Y_7} + \overline{A_7} \cdot \overline{B_7} \cdot Y_7$. For subtraction, it occurs if subtracting a negative from a positive gives a negative result, or vice versa.

1. Single-Cell Memory Working Principle:

A single bit of memory (a Binary Cell) is constructed using a D latch or gated SR latch coupled with logic gates. The cell interface contains three control terminals:

  • Select Input (SEL): Activated by the row decoder to enable the cell for reading or writing.
  • Write Enable (WE): Gated with SEL. When $SEL=1$ and $WE=1$, input data on the data line is latched into the cell.
  • Read/Output Enable: Controls the connection of the internal latch output $Q$ to the output Bit Line via a tri-state buffer. When $SEL=1$ and $WE=0$, the buffer is active and drives the stored data onto the output line.

Inside the cell, the logic structure ensures that the latch's enable terminal $E = SEL \cdot WE$. Output $Q$ is driven to Data Out through a tri-state buffer: $DataOut = (SEL \cdot \overline{WE}) ? Q : \text{High-Z}$.

2. Structural Layout of the 4x4 RAM Array:

To arrange 16 bits of storage as a 4-word by 4-bit array, the cells are structured in a 2D matrix (4 rows $\times$ 4 columns):

  • Address Decoder: A 2-to-4 active-high binary decoder takes address lines $A_1 A_0$ to generate four Word Lines ($WL_0$ to $WL_3$). Each Word Line is connected in parallel to the Select (SEL) pin of all four cells in its respective row.
  • Write Control Lines: The Write Enable ($WE$) signal from the CPU is routed in parallel to all 16 cells in the array.
  • Bit Lines (Data Inputs/Outputs): The array contains four vertical Data Input paths ($D_3$ to $D_0$) and four vertical Data Output paths ($Q_3$ to $Q_0$). Column $j$ cells share input line $D_j$ and output line $Q_j$ via tri-state terminals.

3. Write and Read Operations Flow:

  • Write Operation ($A_1A_0 = \text{address}, WE = 1$): The 2-to-4 decoder activates one Word Line (e.g. $WL_k$). The Select pin of all four cells in row $k$ goes High. Because $WE = 1$, the gate logic inside the cells activates their latch clocks. The values on input lines $D_3 - D_0$ are immediately written into the latches of row $k$. Cells in inactive rows ($WL = 0$) remain unaffected.
  • Read Operation ($A_1A_0 = \text{address}, WE = 0$): The decoder activates Word Line $WL_k$. Since $WE = 0$, the tri-state buffer of all four cells in row $k$ is enabled. Their stored charge values drive the shared vertical column Bit Lines, sending the data out on lines $Q_3 - Q_0$. Cells in all other rows remain disconnected (in High-Z state).

1. Systematic Calculations & Design Steps:

To construct a $1024 \times 32$ memory array using $512 \times 16$ basic memory chips, we calculate the requirements for both word size (bit width) and word capacity (address space):

  • Data Word Size (Bit-Width) Expansion: The target word size is 32 bits, while each chip provides only 16 bits. We must place chips in parallel.
    Target Word Width Chip Word Width
    =
    32 bits 16 bits
    = $2$ chips in parallel. Each row bank requires two chips: Column 0 (Low half) handles bits $D_{15} - D_0$, while Column 1 (High half) handles bits $D_{31} - D_{16}$.
  • Word Capacity (Address Space) Expansion: The target capacity is 1024 words, while each chip holds only 512 words. We must stack chips vertically in banks.
    Target Word Capacity Chip Word Capacity
    =
    1024 words 512 words
    = $2$ stacked banks of chips.
  • Total Basic Chips Required: $\text{Total Chips} = \text{Capacity Expansion} \times \text{Width Expansion} = 2 \text{ rows} \times 2 \text{ columns} = 4 \text{ chips}$.
  • Address Line Mapping:
    • A $1024$-word capacity requires $10$ address lines ($A_9 - A_0$) since $2^{10} = 1024$.
    • Each $512$-word chip requires $9$ address lines ($A_8 - A_0$) since $2^9 = 512$.
    • The lower $9$ address lines ($A_8 - A_0$) are wired in parallel directly to the address pins of all four chips.
    • The MSB address line ($A_9$) is decoded to select between Bank 0 (upper row) and Bank 1 (lower row).
  • Chip Select (CS) Logic Generation:
    • We use a 1-to-2 active-high decoder (or an inverter) fed by address line $A_9$.
    • $\text{Bank Select 0 } (BS_0) = \overline{A_9}$. Selects Bank 0 when address space is $0$ to $511$ ($0\text{x}000$ to $0\text{x}1\text{FF}$). It drives the CS pins of Chip 0 and Chip 1.
    • $\text{Bank Select 1 } (BS_1) = A_9$. Selects Bank 1 when address space is $512$ to $1023$ ($0\text{x}200$ to $0\text{x}3\text{FF}$). It drives the CS pins of Chip 2 and Chip 3.

2. Detailed Read/Write Operations:

  • Data Bus Splitting: The 32-bit CPU Data Bus ($D_{31} - D_0$) is split into two halves. The lower data bus lines ($D_{15} - D_0$) connect to Chip 0 (Bank 0) and Chip 2 (Bank 1). The upper data bus lines ($D_{31} - D_{16}$) connect to Chip 1 (Bank 0) and Chip 3 (Bank 1).
  • Write Cycle ($WE=1$): The CPU drives a 10-bit address onto $A_9 - A_0$ and 32-bit write data onto $D_{31} - D_0$. If $A_9 = 0$, $BS_0$ goes High, enabling Chip 0 and Chip 1. The lower 9 bits ($A_8-A_0$) identify the offset within the bank. Chip 0 writes $D_{15}-D_0$ and Chip 1 writes $D_{31}-D_{16}$ into offset location $A_8-A_0$. Chips 2 and 3 remain disabled ($BS_1=0$).
  • Read Cycle ($WE=0$): The CPU drives the address. If $A_9 = 1$, $BS_1$ goes High, activating Chip 2 and Chip 3. Chip 2 outputs its stored 16-bit word onto data lines $D_{15}-D_0$, and Chip 3 outputs its stored 16-bit word onto data lines $D_{31}-D_{16}$. Together, they form the 32-bit word read by the CPU.

A dual-ported register file containing 8 registers ($R_0 - R_7$) of 8 bits each is designed to allow two read operations and one write operation to proceed concurrently during a single clock cycle. This multi-port functionality is achieved using one of two primary architectural alternatives:

Alternative 1: Monolithic Multi-Ported Register Bank (1 Write, 2 Read Ports)

  • Architecture: Constructed using single-latch or D Flip-Flop registers with integrated dual-routing multiplexer networks. Each of the 8 registers outputs its 8-bit contents in parallel to two independent 8-to-1 multiplexer trees (Mux A and Mux B).
  • Read Operations: The CPU provides two 3-bit read addresses, $\text{Addr}_A$ and $\text{Addr}_B$. Mux A select lines are driven by $\text{Addr}_A$, outputting the selected register onto Bus A. Mux B select lines are driven by $\text{Addr}_B$, outputting the selected register onto Bus B. Both reads are purely combinational and occur simultaneously.
  • Write Operation: Driven by a 3-bit write address ($\text{Addr}_W$), 8-bit input write data ($\text{Data}_W$), and a Write Enable ($\text{WE}$) control signal. A 3-to-8 decoder asserts one of 8 Write Select lines when $WE = 1$. The write is edge-triggered, latching data into the selected register on the rising edge of the system clock.
  • Complexity & Hazards: Highly area-efficient but can suffer from write-during-read hazards. If a register is written to and read from in the same cycle, forwarding bypass gates are required to forward the write data directly to the output ports.

Alternative 2: Replicated Register File (Synchronized Banks)

  • Architecture: Replicates the storage cells by implementing two identical copies of the register bank: Bank A and Bank B. Each bank is a simpler 2-ported design (1 Write, 1 Read).
  • Read Operations: Port A read accesses Bank A; Port B read accesses Bank B. Since the files are separate, no routing congestion occurs between Port A and Port B.
  • Write Operation: To maintain data consistency (coherence) across the replicated structures, every write operation is broadcast to both Bank A and Bank B in parallel. Both banks receive the same $\text{Addr}_W$, $\text{Data}_W$, and $WE$ control signal, updating their respective register slots simultaneously on the active clock edge.
  • Complexity & Hazards: Eliminates read-port routing bottlenecks and avoids read-ports conflict delays, making it highly suitable for high-speed pipelines. However, it doubles the silicon layout area and increases dynamic power consumption due to synchronized duplicate writes.

1. Register-ALU Interconnection Datapath Layout:

The hardware routing between the register file and the ALU consists of a multi-bus path coordinated by multiplexers to select input operands:

  • Register File Ports: Two read registers are selected by instruction fields rs (bits [25-21]) and rt (bits [20-16]). The register file outputs their contents to Bus A (operand 1) and Bus B (operand 2).
  • ALUSrc Multiplexer (2-to-1 Mux): Positioned at the second operand input of the ALU. Mux input 0 is fed by Bus B (Register output). Mux input 1 is fed by the Sign-Extended 16-bit Immediate offset value from the instruction register. Control signal ALUSrc selects between them.
  • ALU Core: Receives Bus A directly as Operand 1, and the ALUSrc Mux output as Operand 2. It performs addition, generating an 8-bit result and status flags.
  • Writeback Bus: The ALU output is routed back to the Register File write data port. The destination register is selected by rd (bits [15-11]) or rt, coordinated by the RegDst Mux, and latched when RegWrite = 1.

3. Step-by-Step Instruction Execution Stages:

  1. Fetch (IF): The instruction is fetched from Code Memory at the location pointed to by the Program Counter (PC) and loaded into the Instruction Register (IR). PC is incremented ($PC \leftarrow PC + 4$).
  2. Decode (ID): The Control Unit parses the opcode. Register file addresses are read: Register R1 is loaded onto Bus A. For R-type, Register R2 is loaded onto Bus B. For I-type, the immediate constant 0x45 is sign-extended.
  3. Execute (EX): The ALUSrc Mux routes either Bus B (R2) or the Immediate constant to the ALU. The ALU performs addition: $\text{ALUOut} = R1 + \text{Operand}_2$.
  4. Writeback (WB): The control signal RegWrite = 1 is asserted. The value on the writeback bus ($\text{ALUOut}$) is written back to Register R0 on the rising clock edge.
