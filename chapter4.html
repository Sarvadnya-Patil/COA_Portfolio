Intel Core i9-13900K Cache Hierarchy

Describe the internal cache memory organization of your allotted processor (Intel Core i9-13900K). Detail the size, association, and distribution of Level 1, Level 2, and Level 3 cache across the Performance cores (P-cores) and Efficient cores (E-cores).
// COA RESOLVED ANSWER

The Intel Core i9-13900K features a hybrid x86 architecture with 24 physical cores (8 P-cores, Raptor Cove; and 16 E-cores, Gracemont) and 32 threads. Its high-speed cache hierarchy is organized as follows:

1. Level 1 (L1) Cache (Private, Split Instruction/Data):

  • 8 Performance Cores (P-cores): Each core has its own dedicated 80 KB L1 cache, split into:
    • 32 KB L1 Data Cache (L1-D) (12-way set-associative)
    • 48 KB L1 Instruction Cache (L1-I) (12-way set-associative)
    Total P-core L1: 8 × 80 KB = 640 KB.
  • 16 Efficient Cores (E-cores): Arranged in 4 clusters of 4 cores. Inside each cluster, each individual core has its own:
    • 32 KB L1 Data Cache (L1-D) (8-way set-associative)
    • 64 KB L1 Instruction Cache (L1-I) (8-way set-associative)
    Total E-core L1: 16 × 96 KB = 1,536 KB = 1.5 MB.
  • Total L1 Cache: 640 KB + 1536 KB = 2.125 MB (2,176 KB).

2. Level 2 (L2) Cache (Dedicated for P-cores, Shared for E-core Clusters):

  • P-cores: Each core has a dedicated, private 2 MB L2 cache (16-way set-associative) to minimize core-to-core latency.
    Total P-core L2: 8 × 2 MB = 16 MB.
  • E-cores: Each of the 4 E-core clusters shares a 4 MB L2 cache (16-way set-associative) among its 4 constituent cores.
    Total E-core L2: 4 × 4 MB = 16 MB.
  • Total L2 Cache: 16 MB + 16 MB = 32 MB.

3. Level 3 (L3) Cache (Globally Shared Smart Cache):

  • A massive 36 MB L3 cache (36-way set-associative) is shared globally across all 8 P-cores and all 4 E-core clusters.
  • Communication is managed via a bidirectional ring bus connecting cores, L3 cache slices, and the system agent. The L3 is non-inclusive, avoiding redundant storage of L2 lines to maximize effective capacity.

Memory Chip Internal Organization

Explain the internal organization of a basic memory chip. Detail how a 16x8 bit cell matrix is structured and trace the operation of select, word, and bit lines during read and write cycles.
// COA RESOLVED ANSWER

A memory chip is organized internally as an array of binary storage cells, where each cell stores one bit of information (0 or 1). For a 16 × 8 Memory Chip (16 words of 8 bits each, totaling 128 bits), the layout is organized as follows:

1. Internal Components:

  • Bit Cell Array: Organized as a grid of 16 rows and 8 columns. Each row corresponds to a single 8-bit memory word.
  • Address Decoder (4-to-16 Decoder): Accepts 4 address lines (A0–A3) and decodes them to activate exactly one of the 16 word lines (W0–W15).
  • Word Lines (W0–W15): Horizontal lines. Activating a word line turns on the access transistors for all 8 bit cells in that row.
  • Bit Lines (b0–b7 & complement b'0–b'7): Vertical lines. They carry data into cells (during Write) or read data out of cells (during Read).
  • Sense/Write Circuits: Positioned at the bottom of the bit line columns. During a Read operation, they amplify the small voltage differences driven by the cells onto the bit lines. During a Write operation, they drive strong voltages to overwrite the cells.
  • Control Lines: Chip Select (CS) enables the chip; Read/Write (R/W) selects the operational cycle.

2. Operational Cycle Traces:

  • Read Operation:
    1. Address lines A3A2A1A0 are driven, and Chip Select (CS) is set to High. The Address Decoder activates the matching Word Line (e.g., W3).
    2. The access transistors for the 8 cells in row 3 are switched ON.
    3. Each cell drives a charge onto its vertical Bit Lines (bi and b'i).
    4. The Sense Amplifiers detect this voltage shift, convert it to standard logic levels, and drive the 8 data lines (D0–D7).
  • Write Operation:
    1. The target address is decoded, activating the target Word Line (e.g., W3). CS is enabled, and the R/W line is driven Low (Write mode).
    2. The input data on lines D0–D7 is driven to the Write circuits.
    3. The Write drivers apply strong voltage differentials onto the vertical Bit Lines (bi and b'i).
    4. These voltages force the storage elements (cross-coupled latches or capacitors) of row 3 cells to flip state, saving the data.

Memory Hierarchy Performance Layout

Explain the memory hierarchy concept in computer systems. Present a layout mapping capacities, typical access times, and relative costs. Highlight the role of temporal and spatial locality in cache performance.
// COA RESOLVED ANSWER

1. Memory Hierarchy Pyramid Layout:

Computer memory is organized as a hierarchy because fast memory is expensive and small, while cheap memory is slow but large. The hierarchy matches performance at different tiers:

Hierarchy Level Typical Capacity Access Time Latency Cost per Bit Technology
1. CPU Registers < 2 KB 0.25 ns – 0.5 ns Highest On-chip D-Flip-Flops
2. L1 Cache 32 KB – 64 KB per core 0.8 ns – 1.2 ns Very High On-chip SRAM (6-transistor cell)
3. L2 Cache 512 KB – 2 MB per core 3 ns – 8 ns High On-chip SRAM
4. L3 Cache 8 MB – 64 MB shared 15 ns – 40 ns Medium-High Shared On-chip SRAM
5. Main Memory (RAM) 8 GB – 64 GB 60 ns – 100 ns Medium-Low Off-chip DRAM (1-transistor 1-capacitor cell)
6. Solid-State Storage (SSD) 500 GB – 4 TB 10 μs – 100 μs Low Non-volatile NAND Flash
7. Magnetic Disk (HDD) 2 TB – 16 TB 5 ms – 15 ms Lowest Mechanical rotating platters

2. Role of Locality of Reference in Cache Performance:

Caches are small and can only hold a tiny fraction of the main memory's contents. They succeed because programs exhibit the **Locality of Reference** principle:

  • Temporal Locality (Locality in Time):

    If a program accesses a memory location once, it is highly likely to access the same location again in the near future.
    Example: Loop counter variables or reuse of instruction blocks in a loop. Caches exploit this by retaining recently accessed blocks in memory.

  • Spatial Locality (Locality in Space):

    If a program accesses a specific memory location, it is highly likely to access nearby locations soon.
    Example: Traversing an array or executing sequential machine instructions. Caches exploit this by fetching an entire **Cache Line (e.g., 64 bytes)** containing surrounding words rather than just fetching the single requested word.

Cache Address Mapping Field Partitioning

Contrast the three cache memory mapping techniques: Direct, Associative, and Set-Associative. Explain how a main memory address is partitioned into Tag, Index, and Block Offset fields for each mapping type.
// COA RESOLVED ANSWER

Cache mapping determines how blocks from the large main memory space are mapped into the smaller cache line array. A main memory address is partitioned differently depending on the mapping scheme:

1. Direct Mapping:

  • Mapping Rule: Each main memory block maps to exactly one specific line in the cache, calculated as: Line = (Block Number) Modulo (Total Cache Lines).
  • Address Partitioning: 
    [ Tag Field ] [ Line Index Field ] [ Word Offset Field ]
  • Field Roles:
    • Word Offset: Identifies the specific word inside the block.
    • Line Index: Selects the specific cache line to check.
    • Tag: Stored alongside the line. The CPU compares this Tag with the address's Tag field to verify if it is a Cache Hit.
  • Pros/Cons: Simplest and cheapest lookup logic, but prone to **conflict misses** when two active memory blocks map to the same line index.

2. Fully Associative Mapping:

  • Mapping Rule: A main memory block can reside in any line of the cache.
  • Address Partitioning: 
    [ Tag Field ] [ Word Offset Field ]
  • Field Roles:
    • Word Offset: Identifies the specific word in the block.
    • Tag: Spans the remaining bits. The CPU must compare the Tag field against all cache line tags simultaneously (using associative comparator hardware).
  • Pros/Cons: Completely avoids conflict misses, but requires expensive comparator circuitry, limiting practical sizes.

3. N-Way Set-Associative Mapping:

  • Mapping Rule: The cache is divided into Sets, each containing N lines. A memory block maps to one specific Set, but can reside in any of the N lines within that set.
  • Address Partitioning: 
    [ Tag Field ] [ Set Index Field ] [ Word Offset Field ]
  • Field Roles:
    • Word Offset: Identifies the specific word in the block.
    • Set Index: Selects the specific Set to check.
    • Tag: The CPU compares the Tag against the N line tags in that Set in parallel.
  • Pros/Cons: The industry standard. Combines the speed and simplicity of direct mapping with the conflict-avoidance of associative mapping.
