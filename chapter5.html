5-Stage Branch-Taken Cycle Sizing

Consider an instruction pipeline with five stages without any branch prediction: Fetch Instruction (FI), Decode Instruction (DI), Fetch Operand (FO), Execute Instruction (EI) and Write Operand (WO). The stage delays for FI, DI, FO, EI and WO are 5ns, 7ns, 10ns, 8ns, 6ns respectively. There are intermediate storage buffers after each stage and the delay of each buffer is 1 ns. A program consisting of 12 instructions I1, I2, I3 ....... I12 is executed in this pipelined processor. Instruction I4 is the only branch instruction and its branch target is I9. If the branch is taken during the execution of this program, calculate the time (ns) needed to complete the program.
// COA RESOLVED ANSWER
Given: Pipeline Stages: FI = 5 ns DI = 7 ns FO = 10 ns EI = 8 ns WO = 6 ns Buffer delay = 1 ns 1) Clock cycle time: Tcycle = max(5, 7, 10, 8, 6) + 1 Tcycle = 10 + 1 = 11 ns // 2) Taken Branch Execution: Branch instruction = I4 Branch target = I9 Since the branch is taken, the executed instructions are: I1, I2, I3, I4, I9, I10, I11, I12 Total executed instructions (n) = 8 For a 5-Stage pipeline (k = 5): Total cycles = k + n - 1 = 5 + 8 - 1 = 12 cycles Branch penalty = 3 cycles ∴ Total cycles = 12 + 3 = 15 cycles 3) Execution time: Execution time = Total Cycles × Tcycle = 15 × 11 = 165 ns ∴ Execution time = 165 ns

Data Dependency Hazard Identification

Consider the following set of instructions and identify if hazards occur. Draw a neat timing diagram to illustrate the pipeline stalls that occur due to these hazards.
Set A: A ← 3 + A,   B ← 4 × A
Set B: A ← 5 × C,   B ← 20 + C
Set C: Mul R4, R3, R2,   Add R6, R4, R5
// COA RESOLVED ANSWER
1) Analysis of Set A: A ← 3 + A, B ← 4 × A - **Hazard**: RAW hazard on variable A because B ← 4 × A needs the updated value of A before A ← 3 + A writes it back. - **Stalls**: Without forwarding, Instruction 2 stalls in Decode for 2 cycles until Instruction 1 writes back A.
Instruction 1 2 3 4 5 6 7 8
I1: A ← 3 + A F1 D1 FO1 E1 W1
I2: B ← 4 × A F2 D2 STALL STALL FO2 E2 W2
2) Analysis of Set B: A ← 5 × C, B ← 20 + C - **Hazard**: None. Both instructions read variable C and write to separate destination variables (A and B). Since there are no dependencies, no stalls or bubbles are introduced. 3) Analysis of Set C: MUL R4, R3, R2, ADD R6, R4, R5 - **Hazard**: RAW hazard because ADD (I2) uses register R4 produced by MUL (I1). The instruction I2 needs the result before I1 writes it back. - **Pipeline Diagram (Without Forwarding):**

Assume that the Decode stage allows two Decode steps to proceed in parallel. Show that the delay bubble can be completely eliminated if the register file allows two Write steps to proceed in parallel.
// COA RESOLVED ANSWER
Instruction I2 depends on the result of I1, so I2 stalls in decode stage. This blocks I3 and I4, creating a delay bubble. Original Pipeline:
Instruction 1 2 3 4 5 6 7 8 9
I1 F D E W
I2 F D STALL D E W
I3 F " STALL D E W
I4 F D E W

Instruction 1 2 3 4 5 6 7 8 9

Slowest Stage Cycle Sizing with Buffer Overheads

Consider an instruction pipeline with five stages without any branch prediction: Fetch Instruction (FI), Decode Instruction (DI), Fetch Operand (FO), Execute Instruction (EI) and Write Operand (WO). The stage delays for FI, DI, FO, EI and WO are 15ns, 4ns, 15ns, 5ns, 15ns respectively. There are intermediate storage buffers after each stage and the delay of each buffer is 4 ns. A program consisting of 10 instructions I1, I2, I3 ....... I10 is executed in this pipelined processor. Instruction I2 is the only branch instruction and its branch target is I9. If the branch is taken during the execution of this program, calculate the time (ns) needed to complete the program.
// COA RESOLVED ANSWER
Given: Stage - Delay: FI = 15 ns DI = 4 ns FO = 15 ns EI = 5 ns WO = 15 ns Buffer delay = 4 ns Clock cycle: T = max(15, 4, 15, 5, 15) + 4 = 15 + 4 T = 19 ns Branch instruction = I2 Target = I9 Executed Instructions: I1, I2, I9, I10 Total instructions executed = 4 Pipeline stages = 5 Base cycles = 5 + 4 - 1 = 8 Branch penalty = 3 cycles Total cycles = 8 + 3 = 11 Execution time = 11 × 19 = 209 ns ∴ Execution time = 209 ns//
