What is COA?

Computer Organization and Architecture (COA) describes the operational capabilities and logical design of computer hardware. While Computer Architecture defines the programmer's view of the machine—such as instruction formats, addressing modes, and registers—Computer Organization deals with the physical hardware details and control paths that realize the architecture, including functional execution units (ALUs), memory structures, and interconnection buses.

About the Course

The Computer Organization & Architecture (25ECSF102) course at KLE Technological University bridges the gap between high-level language execution and logical hardware gate realities. Through this course, students master CPU datapath structures, hardware controls, instruction pipeline timing grids, memory address partitioning, and the physical interconnections that allow instructions to execute in physical space.

Processor Specifications

A modern processor executes program instructions by orchestrating ALU, memory, and register pathways. Exploring clock rates, word size widths, cache sizing, and ISA designs is essential to evaluate structural limits.

Von Neumann vs. Harvard

Von Neumann architectures utilize a single physical bus path for instruction and data fetches, leading to the bus bottleneck. Harvard systems separate code and data paths physically (using split caches L1i/L1d), achieving parallel fetching.

Determining execution time limits involves analyzing CPU clock rates, CPI (Cycles Per Instruction), and final code count. Benchmarking allows designers to rank processors objectively.

Mathematical Metrics

Amdahl's Law models theoretical speedup limits by dividing programs into serial and parallel portions. Speedup is bound by the serial fraction.

// CPU Performance Arithmetic:

Amdahl's Formula: Speedup = 1 / ((1 - P) + (P / N))

Hardware Architecture Datapath (5-Stage CPU)

This multi-stage execution architecture divides instructions into isolated hardware phases: **Instruction Fetch**, **Decode/Reg Read**, **Execute/ALU**, **Memory Access**, and **Writeback**. This partitioning maximizes performance and allows for overlapping pipelined execution.

// 5-Stage Pipeline Execution State:
Stage 1 (IF):   Fetch instruction word from PC address
Stage 2 (ID):   Read registers Address A & B into RA & RB
Stage 3 (EX):   ALU execution (opcodes, offsets, MuxB)
Stage 4 (MEM):  Read/Write DRAM caches using RZ address
Stage 5 (WB):   Write RY result back to port C in Register File

Memory Hierarchy & Cache Organisation

No single memory is fast, large, and cheap simultaneously. The hierarchy exploits locality to give programs the illusion of infinite fast memory.

Hierarchy & Locality

Temporal locality — recently used data will be used again. Spatial locality — nearby data will be used soon. These principles make the hierarchy effective.

// Memory Hierarchy (fast -> slow)
Registers ~0.3ns ~1 KB #####
L1 Cache ~1ns ~32 KB ####
L2 Cache ~4ns ~256 KB ###
L3 Cache ~10ns ~8 MB ##
DRAM ~60ns ~16 GB #
SSD ~100µs ~1 TB c
HDD ~10ms ~10 TB c

// Cache mapping
Direct Mapped: 1 possible cache location
Set Assoc: k possible locations (k-way)
Fully Assoc: any cache location
Cache Hit: Data in cache → served in ~1 ns. Cache Miss → fetch from DRAM → 60x penalty.
Cache Line: 64 bytes. Loading one word loads the entire line — exploiting spatial locality.
Write Policies: Write-through (memory always current) vs Write-back (update on eviction only).
Replacement: LRU (Least Recently Used), FIFO, or Random — decides which line to eviction a miss.

Memory Chip Organization (16x8 Bit Cell Matrix)

Inside modern semiconductor memory ICs, data is stored in a structured **2D grid of memory cells**. An **Address Decoder** decodes binary address lines (A0 - A3) to activate exactly one **word line** (W0 - W15). Differential **bit lines** (bi, b'i) carry read charges down to **Sense/Write Circuits** or drive write values.

Address Decoder: Decodes address inputs to activate one specific row word line.
Word Line (Select): Acts as a row enable signal, opening access gates for all cells in that row.
Differential Bit Lines: High-speed column buses carrying bit values (bi) and complements (b'i).

I/O Organization & Parallel Computing

Input/Output structures link physical world peripherals with internal execution cores. Parallel processing multiplies performance by distributing tasks across multiple concurrent computing threads.

Data Transfer Techniques

Three key methods orchestrate data transfers: Programmed I/O (CPU polls continuously), Interrupt-Driven I/O (device alerts CPU via interrupt vector lines when ready), and DMA (Direct Memory Access) (specialized controllers transfer blocks directly to RAM, freeing the CPU).
